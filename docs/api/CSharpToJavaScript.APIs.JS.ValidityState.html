TypeMismatch

The read-only typeMismatch property of a ValidityStatehttps://developer.mozilla.org/en-US/docs/Web/API/ValidityState object indicates if the value of an {{HTMLElement(&quot;input&quot;)}}, after having been edited by the user, does not conform to the constraints set by the element&apos;s typehttps://developer.mozilla.org/en-US/docs/Web/HTML/Element/input#input_types attribute.

public bool TypeMismatch { get; }

Property Value

bool

Remarks

If the type attribute expects specific strings, such as the {{HTMLElement(&quot;input/email&quot;, &quot;email&quot;)}} and {{HTMLElement(&quot;input/url&quot;, &quot;url&quot;)}} types and the value doesn&apos;t conform to the constraints set by the type, the typeMismatch property will be true.

The {{HTMLElement(&quot;input/email&quot;, &quot;email&quot;)}} input type expects one or more valid email addresses, depending on whether the multiplehttps://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/multiple attribute is present. A valid email address includes an email prefix and a domain, with or without a top level domain. If the value of the email input is not an empty string, a single valid e-mail address, or one or more comma separated email address if the multiplehttps://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/multiple attribute is present, there is a typeMismatch.

The {{HTMLElement(&quot;input/url&quot;, &quot;url&quot;)}} input type expects one or more valid URLs, depending on whether the multiplehttps://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/multiple attribute is present. A valid URL includes a protocol, optionally with an IP address, or an optional subdomain, domain, and top level domain combination. If the value of the URL input is not an empty string, a single valid URL, or one or more comma separated URLS if the multiplehttps://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/multiple attribute is present, there is a typeMismatch.
